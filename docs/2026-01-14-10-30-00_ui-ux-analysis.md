## 2. Problemi Identificati

### 2.1 Accessibilità (WCAG 2.1)

#### Critico - Contrasto Colori

```
Problema: Giallo (#fecd39) su bianco ha contrasto ~2.1:1
Requisito WCAG AA: Minimo 4.5:1 per testo normale, 3:1 per testo grande
Impatto: Utenti ipovedenti non possono leggere il testo
```

**Soluzione proposta:**
```css
/* Giallo accessibile per testo */
--pavesa-yellow-accessible: #b38f00; /* 4.5:1 su bianco */
--pavesa-yellow-dark: #8a6e00;       /* 7:1 su bianco */
```

#### Alto - Navigazione Keyboard

| Problema | Soluzione |
|----------|-----------|
| Nessun skip-to-main link | Creare `SkipLink.astro` |
| Focus states invisibili | Definire `:focus-visible` globale |
| Menu mobile senza `aria-expanded` | Aggiungere attributi ARIA |

#### Alto - Carosello

| Problema | Soluzione |
|----------|-----------|
| Nessun `aria-live` per annunci | Aggiungere region live |
| Nessun `aria-current` per slide attiva | Implementare stato |
| Nessun controllo pausa | Aggiungere pulsante pause/play |
| Nessuna navigazione keyboard | Tab navigation tra dots |

#### Medio - Motion Sensitivity

```css
/* Mancante - da aggiungere */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 2.2 Performance

| Problema | Impatto | Soluzione |

### 2.3 Type Safety (TypeScript)

| Problema | Soluzione |

### 2.4 UX Patterns

#### i18n Migliorato
```typescript
export type TranslationKey = keyof typeof ui[typeof defaultLang];

export function useTranslations(lang: Locale): (key: TranslationKey) => string {
  return (key) => ui[lang][key] ?? ui[defaultLang][key];
}

export function getLocalizedPath(path: string, lang: Locale): string {
  if (lang === defaultLang) return path;
  return `/${lang}${path}`;
}
```
